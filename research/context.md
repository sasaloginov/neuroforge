# Research: Тест ask_owner flow

## Суть задачи

Тестовая задача для проверки механизма `ask_owner` — возможности агента остановить выполнение, задать вопрос владельцу задачи через MCP tool и продолжить работу после получения ответа.

## Что проверялось

1. Аналитик получает задачу
2. Аналитик вызывает `mcp__neuroforge__ask_question` с вопросом: «Какой цвет предпочесть для кнопки — синий или зелёный?»
3. Выполнение приостанавливается до получения ответа
4. Владелец отвечает: **«Синий»**
5. Аналитик продолжает работу с учётом ответа

## Затрагиваемые файлы

Нет. Тестовая задача — проверка оркестрации, изменения в коде не требуются.

## Результат

Механизм ask_owner работает корректно: вопрос доставлен, ответ получен, работа продолжена.
